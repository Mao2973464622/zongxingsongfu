全案纪要（面向开发者）

📌 项目概况

- 主题：端午安康 · 祈福趣味
- 形态：单屏互动小站（粽子 + 点击祝福 + 摇一摇换表情 + 长按生成卡）
- 技术栈：纯 HTML5 + CSS3 + Vanilla JS，单文件零依赖
- 体积：< 30KB

🗺️ 设计演变：从5屏到1屏

初始方案（5屏）

1. 龙舟竞渡（划桨入场）
2. 粽星捧月（6个拟人粽子）
3. 艾草飘香（长按祈福签）
4. 五色丝线（编织五彩绳）
5. 安康寄语（生成海报）

问题

- 首屏强制交互 → 跳出率高
- 5屏过长，用户停留 <30s
- 功能堆砌，缺少“即时快乐”

最终决策

砍掉4屏，只留 一个粽子，核心交互压缩为：点 → 摇 → 长按。

🧩 最终功能

操作 反馈
点击粽子 弹跳 + 祝福语气泡（2.5s消失）
摇一摇手机 4种表情循环切换（桌面端随机降级）
长按 2.6s 弹出祈福卡（截屏分享）
背景 12片SVG艾草叶飘落

⚙️ 技术要点

- 祝福不重复：
"usedIndices[]" 洗牌算法
- 摇一摇：
"devicemotion" 加速度差值阈值 >42
- 长按：
"setTimeout" 2600ms
- 兼容：
"click" + 
"touchend" 双绑定，
"clamp()" 响应式

🔴 关键决策

- 放弃 
"html2canvas" → 纯 DOM 卡片 + 截屏引导（零依赖）
- 书法字体仅用于标题，正文用系统圆体（加载快）
- 桌面端无陀螺仪 → 15% 概率随机换表情

🧠 三条铁律

1. 第1秒法则：打开就要有东西可点
2. 减法 = 设计：砍掉多余的屏，聚焦核心10%
3. 零依赖是特性：纯原生，任何人任何设备都能打开
